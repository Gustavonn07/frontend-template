# Accordion

O **Accordion** é um componente que permite **expandir e colapsar seções de conteúdo**.

Ele organiza informações em blocos independentes chamados **items**, onde cada item possui:

* um **trigger** (botão que controla o estado)
* um **content** (conteúdo exibido quando aberto)

O estado de abertura é controlado pelo componente `Accordion`, que define **como os itens se comportam** (um aberto ou vários abertos).

---

# Como funciona

O Accordion trabalha com **controle de estado interno baseado em valores (`value`)**.

Cada `AccordionItem` possui um identificador:

```tsx
value="item-1"
```

O componente `Accordion` utiliza esses valores para determinar **quais itens estão abertos**.

Existem dois modos de funcionamento:

### `single`

Apenas **um item pode ficar aberto por vez**.

Abrir um novo item fecha o anterior.

### `multiple`

Permite que **vários itens permaneçam abertos simultaneamente**.

---

# Estrutura básica

A estrutura mínima do Accordion é composta por quatro componentes:

```tsx
<Accordion>
  <AccordionItem>
    <AccordionTrigger />
    <AccordionContent />
  </AccordionItem>
</Accordion>
```

### Hierarquia

```
Accordion
 └ AccordionItem
    ├ AccordionTrigger
    └ AccordionContent
```

# Tipos principais

Alguns tipos importantes usados pelo Accordion:

```ts
type AccordionType = "single" | "multiple"
```

Quando o tipo é `"single"`:

```ts
value: string
```

Quando o tipo é `"multiple"`:

```ts
value: string[]
```

Isso permite controlar o componente de forma **controlada ou não controlada**.

# Controle de estado (modo controlado)

O Accordion também pode ser controlado externamente:

```tsx
const [value, setValue] = useState("item-1")

<Accordion
  type="single"
  value={value}
  onValueChange={setValue}
>

  <AccordionItem value="item-1">
    <AccordionTrigger>Section</AccordionTrigger>
    <AccordionContent>Content</AccordionContent>
  </AccordionItem>

</Accordion>
```

Isso é útil quando o estado precisa ser sincronizado com:

* formulários
* URL
* estado global
* lógica de aplicação

---

Ele pode funcionar em dois modos:

* `single`
* `multiple`

E suporta **controle interno ou externo de estado**.
